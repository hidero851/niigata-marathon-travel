# 新潟マラソンナビ 設計書

**最終更新：2026-05-09**

---

## 1. サービス概要

### コンセプト
新潟県内のマラソン大会をきっかけに、宿泊・食・観光をまとめて探せる地域体験プラットフォーム。  
マラソン参加者が「大会に出る → 宿泊を探す → 旅を楽しむ」という流れを一つのサイトで完結できることを目指す。

### キャッチコピー
> 走る旅で、新潟をもっと好きになる。

### 現在のフェーズ
MVP開発完了・商用公開準備中（2026年5月20日公開予定）。  
大会情報・宿泊・特産品は仮データ。正式公開前に各大会の実データ差し替えが必要。

## 2. 技術スタック

| 分類 | 技術 |
|---|---|
| フレームワーク | React 18 + TypeScript |
| ビルドツール | Vite |
| スタイリング | Tailwind CSS（カスタムカラー navy-* を追加） |
| ルーティング | React Router v6 |
| アイコン | lucide-react |
| SEO | react-helmet-async（OGP・Twitterカード・メタタグ） |
| 認証 | Supabase Auth（signInWithPassword） |
| データ永続化 | localStorage + Supabase DB（デュアルライト方式） |
| ホスティング | Vercel（Hobby プラン・無料） |
| バージョン管理 | Git / GitHub（hidero851/niigata-marathon-travel） |
| カスタムドメイン | marathon-navi.com（お名前.com + Vercel DNS連携） |

### Supabase 利用方針
- **Auth**：管理画面ログインに使用（`signInWithPassword`）
- **DB**：`admin_settings` テーブル（KVストア形式）に管理設定を保存
- **デュアルライト**：書き込み時は localStorage + Supabase DB の両方に保存。読み込みはlocalStorageをキャッシュとして使用し、起動時に `loadFromSupabase()` でSupabaseからlocalStorageへ同期

### 4-1. トップページ（`/`）
| セクション | 内容 |
|---|---|
| ヒーロー | キャッチコピー・地域／月／距離の検索フォーム・タグ一覧 |
| エントリーアラート | エントリー開始・締切が近い大会をカード表示（設定がなければ非表示） |
| How It Works | 3ステップのコンセプト説明 |
| 注目の大会 | おすすめ大会3件（管理画面設定優先、未設定時はコード固定） |
| 魅力で探す | タグ一覧ボタン |
| 免責 | 仮データ注意文 |

**おすすめ大会（コード固定デフォルト）**
```
1. 高田城ロードレース（takada-castle-road-race）
2. 南魚沼グルメマラソン（minamiuonuma-gourmet-marathon）
3. 佐渡トキマラソン（sado-toki-marathon）
```
※ 管理画面でおすすめ設定した場合はそちらが優先。

**エントリーアラート（EntryAlertSection）**
- 「⏰ もうすぐエントリー開始」（オレンジ）
- 「🔴 もうすぐエントリー締切」（レッド）
- 残日数に応じて色が変化：赤（3日以内）→オレンジ（7日以内）→グレー
- アラート表示日数は管理画面で設定可能（デフォルト14日）
- 対象大会がなければセクション全体が非表示

### 4-3. 大会詳細ページ（`/events/:id`）
セクション順序（上から）：

| # | セクション | 内容 |
|---|---|---|
| ① | ヒーロー | 大会名・場所・日付・タグ・CTAボタン |
| ② | 大会基本情報 | 距離・日程・参加費・定員等のテーブル＋**前泊推奨日カード**（eventDate設定時のみ） |
| ③ | このレースで楽しめること | ハイライト画像グリッド |
| ④ | おすすめ参加プラン | モデルプラン（当日スケジュール）＋楽天CTA |
| ⑤ | 宿泊エリア | 🥇🥈🥉の3エリアカード＋楽天CTA |
| ⑥ | エントリー誘導（緑） | 「宿泊の手配が終わったら…」→エントリーリンク（entryUrl設定時のみ） |
| ⑦ | お土産・特産 | 特産品カードグリッド |
| ⑧ | 注意バナー | 宿泊早期予約の注意 |
| ⑨ | 最終CTA | 楽天トラベルCTA＋エントリーリンク |

**スティッキーバナー（画面下部固定）**  
宿泊セクションが画面内に見えていない間、「この大会の宿泊を今すぐ予約」バナーを表示。  
宿泊セクションが見えたら自動で非表示（IntersectionObserver 使用）。

### 4-7. 管理ログイン（`/admin/login`）
- Supabase Auth によるメール・パスワード認証
- 認証成功後に `/admin` へリダイレクト
- 未認証状態で `/admin` にアクセスした場合も `/admin/login` にリダイレクト

### 4-8. 管理画面（`/admin`）
認証済みユーザーのみアクセス可能。3つのタブで構成。

**タブ1：おすすめ大会設定**
- トップの「注目の大会」に表示する大会・順序を指定

**タブ2：大会管理**（サブタブで機能選択）

| サブタブ | 機能 |
|---|---|
| ビジュアル | ヒーロー画像・キャッチコピー・開催日・公式URL・ハイライト等を上書き |
| 宿泊エリア | 大会詳細の宿泊エリア情報を管理画面から上書き |
| 参加プラン | モデルプラン（当日スケジュール）を管理画面から上書き |
| 特産品設定 | 特産品のビジュアル設定 |
| 特産品紐づけ | 大会と既存特産品の紐付けを設定 |
| 独自特産品 | 管理画面からのみ追加できる特産品を登録 |
| エントリー日程 | 大会ごとのエントリー開始日・締切日を設定＋アラート表示日数設定 |

**大会の新規作成**
- 管理画面から新しい大会データを作成できる
- 作成時は「下書き」状態（公開されない）
- プレビューボタン（Eye）で非公開のまま詳細ページを確認可能
- 公開ボタン（Globe）で「公開中」に切り替え
- 公開済みの大会は一般ユーザーに表示される

**タブ3：データ分析**
- アクセスログ・クリックログの集計
- 管理設定の一括リセット

## 5. データ構造

### MarathonEvent（大会）
```typescript
{
  id: string               // 一意のID（例: 'takada-castle-road-race'）
  name: string             // 大会名
  location: string         // 開催地（例: '上越市'）
  prefecture: string       // 都道府県
  date: string             // 表示用日付（例: '2026年6月7日（日）予定'）
  eventDate?: string       // ISO日付（例: '2026-06-07'）← 前泊推奨日の計算に使用
  month: string            // 開催月（フィルタ用）
  distances: string[]      // 種目（例: ['ハーフ', '10km', '5km']）
  catchCopy: string        // キャッチコピー
  tags: string[]           // タグ一覧
  entryUrl: string         // エントリーURL（RUNNET等）
  officialUrl: string      // 公式サイトURL
  fee: string              // 参加費
  capacity: string         // 定員
  draft?: boolean          // true = 下書き（管理画面作成イベントのみ）
  highlights: EventHighlight[]    // このレースで楽しめること
  accommodations: Accommodation[] // 宿泊エリア情報
  localProducts: LocalProduct[]   // 関連特産品
  modelPlans: ModelPlan[]         // 参加モデルプラン
  sourceInfo: SourceInfo[]        // 出典情報
}
```

### EntryDateEntry（エントリー日程）
```typescript
{
  eventId: string           // 大会ID
  entryStartDate?: string   // エントリー開始日（ISO形式）
  entryEndDate?: string     // エントリー締切日（ISO形式）
}
```

### admin_settings テーブル（Supabase DB）
| id（キー） | value | 用途 |
|---|---|---|
| featuredEventSettings | FeaturedEventSetting[] | おすすめ大会設定 |
| eventVisualSettings | EventVisualSetting[] | ビジュアル設定 |
| adminCreatedEvents | MarathonEvent[] | 管理画面作成イベント |
| hiddenEventIds | string[] | 非表示イベントID |
| eventProductAssignments | EventProductAssignment[] | 特産品紐づけ |
| eventEntryDates | EntryDateEntry[] | エントリー日程 |
| entryAlertDays | number | アラート表示日数 |

## 9. 現状の制約と課題

| 課題 | 内容 | 対応方針 |
|---|---|---|
| 全データが仮データ | 大会情報・宿泊・特産品すべて仮 | 正式公開前に実データへ差し替え |
| eventDate未設定が多い | 前泊推奨日カードが高田城以外で非表示 | 各大会のデータに `eventDate` を追記 |
| entryUrl未設定が多い | エントリー誘導カードが高田城以外で非表示 | 各大会の実URLを調査して設定 |
| アフィリエイトURL共通 | 全大会・全CTAが同じURLを使用 | 将来的に地名・日程パラメータで最適化 |
| Vercel Hobby の制約 | 商用利用時の規約確認が必要 | 必要に応じて Pro プランへ移行 |

## 11. 開発メモ

- `npm run dev` → 開発サーバー起動（localhost:5173）
- `npm run build` → 本番ビルド（エラーチェック兼用）
- `git push` → GitHub 経由で Vercel に自動デプロイ
- データ変更 → `src/data/niigata-events-draft.ts` を編集して push
- おすすめ大会変更 → 管理画面の「おすすめ大会設定」タブ、またはコード内 `DEFAULT_FEATURED_IDS`
- 管理画面ログイン → Supabase Dashboard でユーザー管理
