**最終更新：2026-05-03**

---

## 1. サービス概要

### コンセプト
新潟県内のマラソン大会をきっかけに、宿泊・食・観光をまとめて探せる地域体験プラットフォーム。  
マラソン参加者が「大会に出る → 宿泊を探す → 旅を楽しむ」という流れを一つのサイトで完結できることを目指す。

### キャッチコピー
> 走る旅で、新潟をもっと好きになる。

### 現在のフェーズ
MVP（最小限の製品）開発中。データはすべて仮データ。正式公開前に各大会・宿泊・特産品の情報を実データに差し替える必要がある。

## 4. 画面・ページ一覧

### 4-1. トップページ（`/`）

**おすすめ大会（コード固定）**


### 4-2. 大会一覧ページ（`/events`）
- 地域・月・距離・タグでフィルタリング可能
- URLパラメータ（`?region=` `?month=` `?tag=` 等）でフィルタ状態を共有できる

---

### 4-3. 大会詳細ページ（`/events/:id`）
セクション順序（上から）：

| # | セクション | 内容 |
|---|---|---|
| ① | ヒーロー | 大会名・場所・日付・タグ・CTAボタン |
| ② | 大会基本情報 | 距離・日程・参加費・定員等のテーブル＋**前泊推奨日カード**（eventDate設定時のみ） |
| ③ | このレースで楽しめること | ハイライト画像グリッド |
| ④ | おすすめ参加プラン | モデルプラン（当日スケジュール）＋楽天CTA |
| ⑤ | 宿泊エリア | 🥇🥈🥉の3エリアカード＋楽天CTA |
| ⑥ | エントリー誘導（緑） | 「宿泊の手配が終わったら…」→エントリーリンク（entryUrl設定時のみ） |
| ⑦ | お土産・特産 | 特産品カードグリッド |
| ⑧ | 注意バナー | 宿泊早期予約の注意 |
| ⑨ | 最終CTA | 楽天トラベルCTA＋エントリーリンク |

**スティッキーバナー（画面下部固定）**  
宿泊セクションが画面内に見えていない間、「この大会の宿泊を今すぐ予約」バナーを表示。  
宿泊セクションが見えたら自動で非表示（IntersectionObserver 使用）。

---

### 4-4. 特産品詳細ページ（`/products/:id`）
- 特産品の詳細情報・購入先・関連大会

### 4-5. データ確認ページ（`/datasource`）
- 全データの出典・利用規約確認状況を一覧表示
- 正式公開前のチェック用

### 4-6. 管理画面（`/admin`）
- URLを知っている人のみアクセス可能（認証なし）
- 下記の設定が可能

| 機能 | 説明 | 保存先 |
|---|---|---|
| おすすめ大会設定 | トップの注目大会を指定 | localStorage |
| 大会ビジュアル設定 | ヒーロー画像・キャッチコピー・開催日・公式URL等を上書き | localStorage |
| 大会非表示 | 静的データの大会を一覧から隠す | localStorage |
| 大会新規作成 | 管理画面からのみ登録できる大会を追加 | localStorage |
| 特産品設定 | 大会と特産品の紐付け | localStorage |

⚠️ **localStorage の制約**：設定はブラウザ・デバイスごとに保存されるため、別のブラウザや別のPCには反映されない。本番公開後の設定変更はコードを修正して git push するのが確実。

## 5. データ構造

### MarathonEvent（大会）
```typescript
{
  id: string               // 一意のID（例: 'takada-castle-road-race'）
  name: string             // 大会名
  location: string         // 開催地（例: '上越市'）
  prefecture: string       // 都道府県
  date: string             // 表示用日付（例: '2026年6月7日（日）予定'）
  eventDate?: string       // ISO日付（例: '2026-06-07'）← 前泊推奨日の計算に使用
  month: string            // 開催月（フィルタ用）
  distances: string[]      // 種目（例: ['ハーフ', '10km', '5km']）
  catchCopy: string        // キャッチコピー
  tags: string[]           // タグ一覧
  entryUrl: string         // エントリーURL（RUNNET等）← 設定するとエントリー誘導カード表示
  officialUrl: string      // 公式サイトURL
  fee: string              // 参加費
  capacity: string         // 定員
  highlights: EventHighlight[]    // このレースで楽しめること
  accommodations: Accommodation[] // 宿泊エリア情報
  localProducts: LocalProduct[]   // 関連特産品
  modelPlans: ModelPlan[]         // 参加モデルプラン
  sourceInfo: SourceInfo[]        // 出典情報
}
```

## 8. 現状の制約と課題

| 課題 | 内容 | 対応方針 |
|---|---|---|
| 全データが仮データ | 大会情報・宿泊・特産品すべて仮 | 正式公開前に実データへ差し替え |
| localStorage依存 | 管理設定がブラウザ・デバイスごとに異なる | 将来的にバックエンド（DB）を導入 |
| eventDate未設定が多い | 前泊推奨日カードが高田城以外で非表示 | 各大会のデータに `eventDate` を追記 |
| entryUrl未設定が多い | エントリー誘導カードが高田城以外で非表示 | 各大会の実URLを調査して設定 |
| 管理画面に認証なし | URLを知っていれば誰でもアクセス可能 | 商用化時にパスワード認証等を追加 |
| アフィリエイトURL共通 | 全大会・全CTAが同じURLを使用 | 将来的に地名・日程パラメータで最適化 |
