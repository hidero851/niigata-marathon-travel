SLIDE 05
⚙️ 管理画面 ① ログイン・大会管理
Supabase 認証 ＋ 大会の新規作成・下書き・公開管理
URL: /admin/login → /admin
ログイン（Supabase Auth）
🔑 メール＋パスワード認証
Supabase の signInWithPassword を使用。認証済みの場合は /admin に直接アクセス可能。未認証は /admin/login にリダイレクト。
大会一覧の操作
🆕 新規大会を作成
管理画面のフォームから大会情報を入力して登録。作成直後は「下書き」状態。
👁 プレビュー（Eye ボタン）
下書きのまま詳細ページを別タブで確認できる。大会運営者への事前確認に使用。
🌐 公開トグル（Globe ボタン）
「下書き」⇔「公開中」を切り替え。公開中のみ一般ユーザーに表示される。
ステータスバッジ
下書き 非公開・プレビューのみ
公開中 一般ユーザーに表示
静的データの操作
✏️ 開催日の上書き（鉛筆ボタン）
静的データの大会は開催日のみ管理画面から変更可能。
🗑 非表示（ゴミ箱ボタン）
静的データは削除ではなく非表示扱い。管理画面作成データは完全削除。
データ同期
☁️ Supabase DB へ自動同期
設定変更は localStorage と Supabase DB の両方に保存される。別デバイスからも設定が引き継がれる。
SLIDE 06
⚙️ 管理画面 ② 大会詳細設定（サブタブ）
大会を選んで各種コンテンツを管理画面から上書き設定
URL: /admin（大会管理タブ）
サブタブ できること 効果
ビジュアル ヒーロー画像URL・キャッチコピー・サブタイトル・公式URL・開催日（eventDate）・ハイライトを上書き 大会詳細ページの見た目が即座に変わる
宿泊エリア 大会詳細の宿泊エリア（🥇🥈🥉）の情報・楽天URLを管理画面から上書き コードを変更せずに宿泊情報を更新できる
参加プラン モデルプラン（当日スケジュール）を管理画面から上書き 大会ごとの旅行プランをカスタマイズ
特産品設定 既存特産品のビジュアル（画像・説明文）を上書き 特産品カードの見た目を管理画面から変更
特産品紐づけ 大会詳細ページに表示する既存特産品を選択・設定 大会と特産品の対応関係を動的に管理
独自特産品 管理画面からのみ追加できる特産品を登録 データファイルを編集せずに特産品を追加
エントリー日程 NEW 大会ごとのエントリー開始日・締切日を設定。アラート表示日数（全体設定）も変更可能 トップページのエントリーアラートセクションに反映
📅 エントリー日程の仕組み
アラート表示日数（デフォルト14日）以内にエントリー開始・締切が迫っている大会を、トップページに自動表示。

残3日以内 → 赤
残7日以内 → オレンジ
残14日以内 → グレー
☁️ 全設定がSupabaseに同期
サブタブで保存した設定はすべてSupabase DBに自動保存される。PCを変えても設定が引き継がれる。

Supabase の役割
機能内容

DB（admin_settings テーブル）管理設定のクラウド保存・デバイス間同期

環境の違い
🖥 ローカル（開発用）
URL: http://localhost:5173
起動: デスクトップの「走る旅-起動.bat」
特徴: PCを閉じると止まる。自分のみ閲覧可。
🌐 Vercel（本番・常時稼働）
URL: marathon-navi.com
特徴: 24時間365日稼働。PCの電源に関係なし。世界中からアクセス可能。
🔑 環境変数（Vercel）
VITE_SUPABASE_URL
VITE_SUPABASE_ANON_KEY
※ Vercel の Environment Variables に設定済み
